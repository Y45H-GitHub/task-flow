# FlowTask Project Overview

Welcome to FlowTask, a local-first, highly responsive, and feature-rich personal task management application. This document serves as a senior-level overview of the product vision, its core features, and the problems it was created to solve.

---

## 🎯 The Core Problem

Standard task managers (e.g., Todoist, Microsoft To Do, Google Tasks) suffer from significant context deficits:
1. **No Creation Context**: Standard apps show *when a task is due*, but fail to answer, *"When did I add this task, and why is it here?"* This leads to stale, low-priority tasks cluttering lists indefinitely without explanation.
2. **Poor Meeting Integration**: Users often need to review action items with specific people (e.g., family members, colleagues). Grouping tasks by project or list does not help you prepare for a meeting with a specific person.
3. **No Geographic Activation**: Standard tasks have location alerts based on coordinates, which are power-intensive and hard to trigger reliably in standard web PWAs. They lack context-based groupings (e.g., "Medical Store", "Print Shop") that allow a user to check off multiple tasks when in a specific shopping district.
4. **Disconnection from Logged Events**: Tasks are often generated by conversations or meetings. When a call or sync happens, action items are recorded, but standard task apps keep logs and tasks completely separate.
5. **Physical Assets Loss**: Standard productivity apps focus on actions, not items. Users frequently lose track of where they stored physical documents or devices (e.g., passport, backup drive, car keys).

---

## ✨ Core Value Propositions

FlowTask addresses these deficiencies through a set of specialized features:

### 1. Created Timestamps & Aging Alerts
Every task card explicitly displays a "Created [time] ago" badge alongside the raw ISO timestamp.
- **Created Timestamps**: Visualizes chronological distance (e.g., `Added 2h ago` or `Added 3d ago`).
- **Aging Alert (🔥)**: Automatically flags any task that has been pending for **7+ days**. This immediately exposes stagnant tasks that should either be prioritized, refactored, or deleted.
- **Notes / Why Field**: A dedicated field prompts the user to document *why* they added the task, keeping critical context alive.

### 2. People-Centric View (Meeting Prep)
FlowTask groups all pending tasks by person (`self`, `kj`, `aj`, `plj`, `hj`, or custom added individuals).
- Clicking the **People** tab displays a visual summary of tasks assigned to each contact, sorted by priority.
- A dedicated **"All Clear ✓"** section highlights people who currently have zero outstanding tasks.
- Perfect for quickly scanning all items to discuss before initiating a call or entering a meeting.

### 3. Places View & Geographic Triggers
Tasks can be tagged with place types (e.g., medical, print, bank, grocery, hardware).
- **📡 Near Me? Geolocation**: Features an on-demand browser geolocation query. Tapping this checks the user's current GPS coordinates, queries the Overpass API for matching shop/amenity categories within a 500-meter radius, and notifies the user of nearby tasks they can complete.
- **🗃 Physical Item Locator**: A built-in registry where users can document the location of critical physical items (e.g., `"Passport" -> "Top dresser drawer"`).

### 4. Conversation & Meeting Log
A built-in journal for recording meetings, phone calls, emails, WhatsApp messages, or in-person syncs.
- Each entry captures who, the channel, date, summary, tags, and a list of comma-separated action items.
- Features real-time full-text search across all logs.

### 5. Settings & Utility ("More" Tab)
- **CSV / JSON Portability**: Full data export/import capability to ensure users own their data and can backup/restore state instantaneously.
- **WhatsApp Formatter**: Copies a clean plain-text summary of all pending tasks (grouped by person) to the clipboard, formatted with markdown for quick paste into WhatsApp/Slack.

## 🎨 Design Philosophy
- **Dark Mode by Default**: Tailored to reduce visual fatigue. The primary background uses a deep navy shade (`#0B1120`) paired with a vibrant solid cyan accent (`#38BDF8`).
- **Bento Box Layout**: Layouts utilize glassmorphism style cards with subtle border borders, creating structured, grouped interfaces.
- **Micro-Animations**: Slide-in transitions, modal bottom-sheets, and smooth transitions are built with CSS keyframes and transitions for maximum performance.
- **Typography**: Space Grotesk for headings and display tags, paired with Inter for body text (loaded via Google Fonts CDN).
- **Zero Emojis in Core UI**: Vector line icons from the **Unicons** pack are utilized throughout, providing a consistent, sleek, professional visual design.
